### 3. 实现拖拽文件处理和实时进度

**增强 `DropArea` 组件：**

```python
# 支持的文件类型
支持格式：.pdf, .png, .jpg, .jpeg, .tiff, .bmp

# 拖拽行为

- 多个文件：批量添加

```

**实时进度显示：**

| 进度组件 | 功能 |
|----------|------|
| **总体进度条** | 显示总处理进度（0-100%） |
| **当前文件标签** | 显示正在处理的文件和状态 |
| **状态栏统计** | 文件总数 | 成功数 | 失败数 |
| **日志区域** | 详细处理日志 |

**进度回调流程：**

```python
# 总体进度
self.progress.emit(10, "🚀 开始处理：🧹 去水印 → 🔍 OCR → 📝 WORD")

# 单文件进度
self.file_progress.emit("document.pdf", 50, "⏳ 处理中")

# 完成
self.finished.emit(True, "处理完成", success_count=8, fail_count=1)
```

## 🔧 技术实现

### 核心类

| 类名 | 职责 |
|------|------|
| `WorkerThread` | 后台处理线程，支持进度回调和停止 |
| `DropArea` | 拖拽区域，支持文件和目录 |
| `ProcessingOptionsWidget` | 功能选项面板 |
| `MainWindow` | 主窗口，整合所有功能 |

### 信号和槽

```python
# WorkerThread 信号
progress = pyqtSignal(int, str)           # 总体进度
file_progress = pyqtSignal(str, int, str) # 单文件进度
finished = pyqtSignal(bool, str, int, int) # 完成

# ProcessingOptionsWidget 信号
options_changed = pyqtSignal()            # 选项变更
```

### 数据处理流程

```
用户选择文件
    ↓
设置处理选项（可组合）
    ↓
点击开始
    ↓
WorkerThread 启动
    ↓
ProcessingPipeline 处理
    ├── 去水印（可选）
    ├── OCR（可选）
    └── WORD 转换（可选）
    ↓
实时进度回调
    ↓
完成并记录历史
```
